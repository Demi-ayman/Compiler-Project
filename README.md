# TINY Language Scanner & Parser

A desktop application that implements the **Scanner (Lexer)** and **Parser** for the **TINY programming language**.  
The system generates tokens from TINY source code and builds a full syntax tree using a **recursive-descent parser**.

## ðŸ“Œ Project Overview

This project includes two main components:

### **1. Scanner (Lexer)**
Reads raw TINY code and converts it into a list of `(tokenValue, tokenType)` pairs based on the official TINY language token list.

### **2. Parser**
Consumes the token list generated by the Scanner and validates whether the program follows TINY grammar rules.  
It outputs:
- Syntax validity (Accepted / Not Accepted)
- A complete syntax tree (displayed via GUI or text output)

A GUI layer allows users to input code, generate tokens, load token files, parse code, and visualize syntax trees.

## âœ¨ Features

### **Scanner Features**
- Tokenizes TINY code into `tokenValue, tokenType` pairs
- Supports all TINY tokens:
  - Keywords: `if`, `then`, `end`, `repeat`, `until`, `read`, `write`
  - Operators: `+ - * / < = :=`
  - Symbols: `; ( )`
  - Identifiers and numbers
- Input through GUI text box or external file
- Outputs token list to a file

### **Parser Features**
- Implements **Recursive Descent Parsing**
- Converts grammar to **EBNF** form
- Outputs:
  - Syntax acceptance
  - Complete syntax tree (GUI or text)
- Can parse new code without restarting the application

### **GUI Features**
- Desktop application (.exe)
- Code input area
- Token viewer
- Syntax tree visualization
- File loading support

### **Error Handling (Bonus)**
- Invalid file paths
- Empty inputs
- Incorrect token formatting
- Unsupported symbols or tokens
